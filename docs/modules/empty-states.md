**НЕ входит (явно):**

- Решения PO, ещё не принятые: «События» при двойной роли (организатор + участник чужих клубов),
  история посещённых событий — см. память проекта.
- Пустая ветка Этапа 2 на EventPage («Состав», 0 подтвердивших) — там уже есть счётчик
  «Подтвердили 0»; аудит просил намёк только «под голосованием» (Этап 1).
- Ошибка загрузки «Финансов» без «Повторить» (`OrganizerClubManage.tsx:65-67`) — не empty-state

- Предвыбор типа активности в едином флоу создания (CTA «Создать сбор» открывает флоу
  с шага выбора типа — как «Создать событие» в EventsTab; расширение `useCreateFlowStore`
  пресетом типа — YAGNI, пока случай один).

### Общие правила волны (для всех пунктов)

1. **Роль-развилка — на вызывающем.** `FoxEmpty` получает готовые title/description/CTA.
2. **Ошибка ≠ пустота (урок F5-20).** Пустая заставка — только при `isSuccess` с пустыми
   данными. `isPending` → скелет/спиннер, `isError` → error-вид с «Повторить».
3. **CTA создания** (активностей) — только через `useCreateFlowStore.open()`
   (`frontend/src/store/useCreateFlowStore.ts`): сам флоу и его guard'ы живут в `AppDock`
   (`components/Layout.tsx`). Пульс-подсветку FAB не использовать — забракована PO.
4. **Тон — на «ты»**, без канцелярита, в духе текстов волн 1–2. Слово «Поиск» — с большой буквы
   (см. открытый вопрос №2 про таб «Главная»).
5. **Хаптика** на каждом CTA: `haptic.impact('light')` — как в существующих обработчиках.
6. Номера строк ниже — актуальный код на 2026-07-20; при реализации сверяться с файлом.

### W3-04. Профиль → «Уровень»: тизер при xp=0 + отдельная ошибка

- **Файл:** `frontend/src/pages/ProfilePage.tsx` (строки 194-199), панель —
  `components/profile/GamificationPanel.tsx`.
- **Триггер:** `useMyGamificationQuery()`; сейчас секция скрыта при `xp === 0 && badges
  пусто`, а ошибка загрузки молча выглядит как «данных нет».
- **Роль:** свой профиль (одна роль).
- **Как сейчас:** `{gam && (gam.xp > 0 || gam.badges.length > 0) && <GamificationPanel/>}` —
  новичок вообще не знает, что уровни существуют.
- **Как должно стать.** Никаких новых механик — `GamificationPanel` уже корректно рендерит
  нулевой стейт из реальных данных бэкенда (`GamificationDto` при xp=0: `level=1`,
  `levelName="Гость"`, `nextLevelName="Свой"`, `xpIntoLevel=0`, `xpSpanToNext=50` —
  см. `XpPolicy.kt`: уровни «Гость» → «Свой» → … кривая 50·i²):
  - гейт секции — **по наличию данных**, не по `isSuccess` (stale-while-error): есть
    `gamificationQuery.data` → секция «Уровень» + `GamificationPanel` показываются всегда
    (панель сама покажет «Гость», «0 XP · ур. 1», пустой прогресс-бар,
    `0 / 50 XP до «Свой»` — это и есть тизер-скелет). При ошибке **фонового** рефетча
    поверх кэша остаётся панель с последними данными — плашка ошибки не выскакивает;
  - при `xp === 0 && badges.length === 0` под панелью — строка-пояснение (`rd-cta-hint`):
    **«Это твой старт. XP начисляется за посещённые встречи и оплаченные
    складчины — с ним растут уровни и открываются бейджи.»**
  - данных нет и `gamificationQuery.isError` — секция «Уровень» + компактная плашка
    `rd-glass rd-empty` c `role="alert"`: title **«Не удалось загрузить уровень»**,
    ghost-кнопка **«Повторить»** → `gamificationQuery.refetch()` (зеркало паттерна
    «Интересов», строки 147-164 того же файла);
  - данных ещё нет и запрос грузится — секция не рендерится (как раньше, скелет не нужен:
    секция ниже фолда, появление после загрузки не дёргает вёрстку).

**Acceptance criteria:**
- GIVEN свежий аккаунт (0 XP, 0 бейджей) WHEN открывает «Профиль» THEN видит секцию
  «Уровень»: «Гость», «0 XP · ур. 1», прогресс 0/50 до «Свой», строку-пояснение про XP.
- GIVEN xp > 0 THEN панель как раньше, строки-пояснения нет.
- GIVEN запрос геймификации упал и закэшированных данных нет THEN плашка «Не удалось
  загрузить уровень» + «Повторить» (и это НЕ выглядит как «уровня нет»).
- GIVEN упал фоновый рефетч поверх уже загруженных данных THEN остаётся панель
  с последними данными, плашка не показывается (stale-while-error).

### W3-08. Управление → «Финансы»: честный хинт для бесплатного клуба и нуля участников

- **Файл:** `frontend/src/pages/OrganizerClubManage.tsx`, компонент `FinancesTab`
  (строки 53-86; хинт — строки 80-83).
- **Триггер/данные:** `FinancesTab` получает `club: ClubDetailDto` целиком (вместо прежнего
  `clubId`; id для запроса финансов берётся из `club.id`) + колбэк `onOpenSettings`
  (переключение `activeTab` на `'settings'`).
- **Роль:** менеджер (владелец и со-организатор — таб виден обоим).
- **Как сейчас:** для любого клуба один и тот же хинт: «Оплата подписок идёт напрямую вам…
  Подтверждайте полученные взносы… кнопкой «Взнос получен»» — бесплатному клубу это враньё
  (взносов нет вообще), клубу без участников — бессмысленно.
- **Как должно стать** (сводка «Активных участников» остаётся во всех случаях):
  - `club.subscriptionPrice === 0`: хинт → **«Клуб бесплатный — взносы не собираются,
    поэтому финансовой сводки нет. Захочешь ввести платную подписку — это в «Настройках».»**
    - ghost-CTA: **«Открыть настройки»** → `onOpenSettings()` (переключить таб; не навигация).
  - `subscriptionPrice > 0 && finances.activeMembers === 0`: хинт → **«Пока некому платить —
    активных участников ещё нет. Когда появятся, здесь будет видно, кто оплатил взнос,
    а подтверждать оплату будешь во вкладке «Участники» на странице клуба.»**
  - `subscriptionPrice > 0 && activeMembers > 0`: текущий смысл, но тон на «ты»:
    **«Оплата подписок идёт напрямую тебе, мимо платформы — поэтому суммы здесь
    не считаются. Подтверждай полученные взносы во вкладке «Участники» кнопкой
    «Взнос получен».»**

**Acceptance criteria:**
- GIVEN бесплатный клуб WHEN владелец открывает «Управление → Финансы» THEN видит текст
  про бесплатный клуб + «Открыть настройки»; тап переключает на таб «Настройки» (URL-таб/
  история не ломаются, это внутритабовое переключение как у ручного тапа).
- GIVEN платный клуб, 0 активных участников THEN текст «Пока некому платить…», без CTA.
- GIVEN платный клуб с участниками THEN прежний смысл хинта на «ты».
- Со-организатор видит то же, что владелец (различий по этой вкладке нет).

### W3-09. EventPage: строка-намёк, когда на голосовании 0 откликов

  comingList.length === 0` — свежее событие Этапа 1 без единого голоса.

  под голосованием пустота без объяснения.
- **Как должно стать:** в слоте секции ростера (под блоком голосования) — строка-намёк

**Acceptance criteria:**

- GIVEN участник голосует «Пойду» THEN строка исчезает, появляется «Предварительные
  голоса · 1».
- GIVEN организатор на том же событии THEN видит вариант про чат клуба.
- GIVEN Этап 2 / отменённое событие THEN строки нет (гейт `showVoting`/`!isCancelled`).
- GIVEN responders ещё грузятся или упали THEN строки нет.
