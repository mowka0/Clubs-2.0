**Статус:** реализовано (ветка `feature/discovery-card-v2`). Дизайн залочен PO 2026-07-22.
**Происхождение:** хендофф `docs/backlog/discovery-card-v2-handoff.md`; мокап-истина по

## 1. Что сделано (3 куска)

### A. Карточка клуба v2 (`ClubCard.tsx`)
- Тройка метрик (возраст · участники · вовлечённость) переехала из тела карточки на
  **компактную полку-уголок в нижнем-левом углу обложки**: панель из материала карточки,
  скруглён только правый верхний угол, метрики голым текстом «иконка + значение»
  (не пилюли). Полка рендерится **только когда загружены `facts`** (как раньше трио).
  - возраст: иконка-часы + `{ageDays} дн` (короткое «дн», без склонения)
  - участники: иконка-люди + `{memberCount}` (голое число)
  - вовлечённость: иконка-молния + `{engagementPercent}%`, цвет `--live` (зелёный)
- Градиент-скрим обложки (`rd-cover::after`) **убран**. Чип цены остался — у него своя
  подложка. Верхний бейдж «встреча HH:MM» **снят с обложки** (сталкивался с ценником на
  узких экранах) → заменён «колонкой расписания» в теле (см. ниже; PO 2026-07-22,
  вариант 17 мокапа `13-meeting-corner.html`).
- Тело: `rd-rankpill` «★ Топ-5 в категории» (при `topInCategory`) + нижний ряд
  `rd-brow`: слева название + город с иконкой-пином, справа — **колонка расписания**
  `rd-meet` (вертикальная акцент-черта + «СЕГОДНЯ»/`HH:MM` столбиком), когда у клуба
  событие **сегодня по локальному календарю** (`isToday` — та же семантика, что у полки
  TodayShelf; старое окно «<24ч» `isHappeningSoon` удалено — оно могло показывать
  «встречу» для завтрашнего утра). Старое трио `rd-mrow`/`rd-mstat` удалено (вместе с CSS).
  `rd-meta` — всегда просто город; текст города обрезается многоточием (`rd-meta-city`),
  не толкая колонку времени — конструктивный запас под будущий район.
- Высота обложки 96 → 112px (по мокапу 11 — полка съедает нижние ~30px фото).

### B. Снос плиток «Репутация / В клубах» (`DiscoveryPage.tsx`)
- JSX-блок `rd-stats` удалён вместе с мёртвыми переменными (`hasReputation`,
  `globalScore`, `global`, `reliablePhrase`) и импортами `tierWord`/`clubsPrepositional`.
- `useMyReputationQuery`/`activeCount`/`reputationQuery.error` **оставлены** — питают
  подпись шапки «Состоишь в N клубах».
- CSS `.rd-stats*` **не тронут** — используется в Профиле и «Управлении».
- Репутация живёт в Профиле (там уже есть) — на Discovery она дублировала шапку.

### C. Полка «Встречаются сегодня» (`TodayShelf.tsx`, новый)
- Горизонтальный ряд мини-карточек между чипами категорий и списком клубов.
- **Вход:** уже загруженный список `clubs` из DiscoveryPage (все страницы infinite
  query). Данные — из приходящего `nearestEvent`, бэк не нужен.
- **Фильтр:** `nearestEvent != null && isToday(eventDatetime)` — совпадение локальной
  календарной даты (Y/M/D), не окно «24 часа». Сортировка по времени события ↑.
- Мини-карточка: бейдж `HH:MM` + «сегодня», название (клап 2 строки),
  `{категория} · {N участников}`. Клик → `/clubs/{id}` + haptic light.
- **Скрыта целиком**, когда клубов с событием сегодня нет.
- Респектит фильтры (строится из отфильтрованной выдачи).

## 2. Новые/изменённые общие утилиты

| Что | Где | Зачем |
|---|---|---|
| `CATEGORY_LABELS`, `categoryLabel()` | `utils/categoryLabels.ts` (новый) | Вынесено из `ClubCard`, переиспользуется `TodayShelf`. Дубли в `MyClubsPage`/`InvitePage`/`OrganizerClubManage` остались — консолидация вне скоупа (см. backlog рефакторинга) |
| `isToday(iso, now?)` | `utils/formatters.ts` | Фильтр полки; `now` инжектится для тестов |
| `formatTimeHM(iso)` | `utils/formatters.ts` | `HH:MM` для бейджа встречи; раньше жил приватно в `ClubCard` |
| `--card-solid`, `--shelf-hl` | `styles/brand-theme.css` | Сплошной материал карточки клуба (тёмная `#17171C` / светлая `#FFFFFF`) + блик кромки полки |

### Почему `--card-solid`, а не `--glass-bg`
Полка лежит НА фото обложки и должна его непрозрачно перекрывать тем же цветом, что
тело карточки — читается «футер вырос вверх». `--glass-bg` полупрозрачный, фото
просвечивало бы. `.rd-club-card` переведён с `--glass-bg` на `--card-solid`
(визуально ≈ то же: glass поверх `--bg` выглядел почти так же), `backdrop-filter`
у карточки удалён как бессмысленный при сплошном фоне.

## 3. Данные

Всё с существующих эндпоинтов: `ClubListItemDto` (список) + `ClubCardFactsDto`
(пакет фактов качества). Района/адреса в данных НЕТ — на карточке только город
(залочено; район = отдельная фича с миграцией).

## 4. Известные ограничения (осознанные)

- Полка «сегодня» видит **только загруженные страницы** выдачи (первая = 20 клубов).
  При сотнях клубов в городе увидит не все. Для MVP ок; настоящий разрез «события
  сегодня по городу» — потом, отдельным эндпоинтом.
- Сигнал «встреча сегодня» на карточке и полка TodayShelf используют **один критерий**
  — календарный `isToday` (окно «<24ч» `isHappeningSoon` удалено вместе с верхним
  бейджем обложки).

## 5. Критерии приёмки

- [x] Карточка: три метрики на полке-уголке в нижнем-левом углу обложки, голым текстом
      с иконками, скруглён правый верхний угол, вовлечённость зелёная.
- [x] Обложка без градиента-скрима; цена/★ читаются; верх обложки не перегружен
      (бейдж встречи ушёл в тело — колонка «СЕГОДНЯ · HH:MM» справа от города).
- [x] Полка = материал тела карточки, бесшовна с телом в обеих темах.
- [x] Тело: ★ (если топ) + название + город с пином. Трио из тела ушло.
- [x] Плитки «Репутация / В клубах» отсутствуют; «Состоишь в N клубах» работает.
- [x] Полка «Встречаются сегодня»: появляется при наличии, скрыта иначе, клик ведёт
      на клуб, респектит фильтры.
- [x] `npm test` зелёный; чистый `tsc`/`npm run build`.
- [ ] Ручная проверка PO на реальном устройстве в Telegram (обе темы) — staging.
