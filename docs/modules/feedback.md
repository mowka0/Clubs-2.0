# Обратная связь (баг-репорты)

Статус: реализовано (ветка `feature/feedback-form`).

## Что делаем

Пользователь фиксирует баг или пожелание прямо из приложения: глобальная кнопка «+»
(FAB дока) → пункт «🐞 Сообщить о проблеме» → форма с текстом → отправка. Бэкенд
доставляет сообщение Telegram-ботом в личку аккаунта техподдержки (@clubs_tech_support).

Решение PO 2026-07-23: обратная связь доступна **всем** пользователям, не только
организаторам. Из-за этого меняется поведение «+»: раньше не-организатор получал тост
«Создавать активности могут организаторы клубов», теперь шит открывается всегда:

- **организатор** видит: «Событие», «Сбор», «🐞 Сообщить о проблеме»;
- **не-организатор** видит только «🐞 Сообщить о проблеме» (заголовок шита — «Обратная связь»).

Старый тост-гардрейл удалён — его роль выполняет состав пунктов шита.

## Ограничение Telegram (важно)

Bot API не умеет слать личные сообщения по `@username` — только по числовому chat id,
и только тем, кто **сам начал диалог с ботом** (`/start` или вход в Mini App).
Поэтому саппорт-аккаунт резолвится через **нашу таблицу `users`** по
`telegram_username` (без `@`, регистронезависимо): аккаунту @clubs_tech_support
достаточно один раз открыть Mini App (на проде — через @clubs_v2_bot, на staging —
через @clubs_v2_test_bot), и доставка заработает. Пока аккаунта нет в БД окружения,
эндпоинт честно возвращает 503, фронт показывает «Не удалось отправить».

## API

### POST /api/feedback — отправить сообщение в поддержку

Авторизация: JWT (любой аутентифицированный пользователь).

Запрос:
```json
{ "message": "string, 1..2000, обязателен, не blank",
  "page": "string ≤ 200, опционален — route, с которого открыта форма" }
```

Ответы:
- `204 No Content` — сообщение доставлено в Telegram (отправка синхронная, ответ = факт доставки);
- `400 VALIDATION_ERROR` — пустое/слишком длинное сообщение;
- `503 FEEDBACK_UNAVAILABLE` — саппорт-аккаунт не найден в `users` или Telegram отклонил отправку;
- `429` — превышен rate limit.

Текст DM саппорту (plain text, без parse mode — пользовательский ввод не
интерпретируется как разметка):

```
🐞 Обратная связь

<message>

— Имя Фамилия (@username), tg id <telegram_id>
— Экран: <page>
```

Строка «Экран» опускается, если `page` не передан.

## Rate limiting

Отдельный пул bucket'ов в `RateLimitFilter`: `/api/feedback` — **3 запроса/мин** на ключ
(фильтр работает до аутентификации, поэтому ключ — IP). Защита от спама в личку саппорта.

## Конфигурация

| Переменная | Дефолт | Смысл |
|---|---|---|
| `TELEGRAM_SUPPORT_USERNAME` | `clubs_tech_support` | username саппорт-аккаунта (без `@`), которому бот шлёт баг-репорты |
| `TELEGRAM_SUPPORT_CHAT_ID` | пусто | числовой chat id саппорта (узнать: @userinfobot). Если задан — **приоритетнее** username: отправка напрямую, без резолва по `users`. Закрывает риск угона канала при смене/освобождении username (security-ревью 2026-07-23) |

`application.yml` → `telegram.support-username`; проброшена в `docker-compose.prod.yml`
и задокументирована в `.env.example`.

## Реализация

Backend `com/clubs/feedback/`:
- `FeedbackController` — `POST /api/feedback`, `@Valid` DTO, 204 при успехе;
- `FeedbackService` — резолв саппорта по username, сборка текста, синхронная отправка;
- `dto/SubmitFeedbackRequest`;
- `FeedbackDeliveryException` (`common/exception/`) → 503 в `GlobalExceptionHandler`;
- `NotificationService.trySendDirectMessage(telegramId, text): Boolean` — синхронная
  отправка без фолбэков (в отличие от best-effort `@Async` DM), результат нужен вызывающему;
- `UserRepository.findByTelegramUsername(username)` — регистронезависимый поиск.

Frontend:
- `AppDock` (Layout.tsx) — FAB всегда открывает `CreateActivityFlow`, тост-гардрейл удалён;
- `CreateActivityFlow` — принимает `canCreate`, пункт «Сообщить о проблеме» ведёт на
  `/feedback` (в `state.from` — route, с которого открыт шит);
- `CreateActivityPicker` → `ActivityTypeOptions` — состав пунктов по `canCreate`;
- `FeedbackPage` (`/feedback`, lazy) — textarea со счётчиком (лимит 2000), submit,
  успех = хаптика success + тост + возврат назад;
- `api/feedback.ts` + мутация `queries/feedback.ts`.

## Критерии приёмки

1. Не-организатор: «+» → шит «Обратная связь» с единственным пунктом → форма → отправка →
   тост «Сообщение отправлено», возврат назад; DM пришёл на @clubs_tech_support.
2. Организатор: «+» → три пункта; «Событие»/«Сбор» работают как раньше (включая
   preset клуба со страницы своего клуба); «Сообщить о проблеме» → форма.
3. Пустое сообщение не отправляется (кнопка disabled), >2000 символов не ввести.
4. DM содержит текст, имя, @username (или «без username»), tg id отправителя и экран.
5. Если саппорт-аккаунт не открывал Mini App этого окружения — форма показывает ошибку
   «Не удалось отправить…», 503 в сети.
6. 4-й запрос за минуту → 429.
