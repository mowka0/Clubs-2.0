# Discovery — фильтры: расхождения с PRD §4.1.2

**Статус:** open · **Создано:** 2026-05-12 · **Origin:** post-flight alignment `feature/discovery-redesign`

PRD §4.1.2 «Фильтрация и поиск» описывает более широкий набор фильтров, чем реализовано в UI после brand-редизайна. Гэп фиксируется здесь явно, чтобы не было «молчаливого расхождения». При появлении пользовательской боли — приоритизировать.

## Что РЕАЛИЗОВАНО (после brand-редизайна)

- ✅ **Город** — `<CityPicker>` bottom-sheet (RU/BY/KZ/AM/GE/AE/TR + города), persist в `localStorage`
- ✅ **Категория** — горизонтальный chip-row tabs
- ✅ **Цена подписки** — `<PriceFilter>` bottom-sheet с 5 presets (Любая / Бесплатно / До 1 000 ₽ / 1 000–3 000 ₽ / От 3 000 ₽), pill в meta-row
- ✅ **Текстовый поиск** — input с 300 ms debounce, по name/description

## Что ОТКЛОНЕНО / НЕ ДЕЛАЛИ vs PRD §4.1.2

| PRD | Реализовано? | Комментарий |
|---|---|---|
| Геопозиция автоопределяется при первом запуске | ❌ | Текущий picker строит выбор из hardcoded списка стран/городов. Backend не отдаёт `GET /cities/nearby`. **Решение:** при обращении пользователя — добавить `navigator.geolocation` + reverse-geocoding |
| Тип доступа (Открытые / Закрытые) | ❌ | До редизайна был в старом `ClubFilters` (`accessType`), backend-поле `ClubFilters.accessType` живо. UI убран как «низкоиспользуемый», но API поддерживает. **Решение:** вернуть в виде ещё одной pill рядом с «Цена», если запросят |
| Цена подписки — ползунок диапазона | ⚠️ pres­ets вместо slider'а | PRD просит slider; реализованы пресеты. Presets проще для большого пальца на mobile + быстрее чем тянуть ползунок. **Решение:** оставить пресеты до пользовательского фидбека про «нужны кастомные значения» |
| Размер клуба (до 20 / 20–50 / 50+) | ❌ | Backend `memberLimit` существует; фильтра в `ClubFilters` нет. Низкий приоритет — клубы пока маленькие, фильтр преждевременный |
| День недели встреч | ❌ | Требует поля «регулярность» у клуба, которого пока нет в модели. Большой scope. Отложено до появления event-расписания |
| «Расширенные фильтры под кнопкой Ещё» | ❌ | UI-паттерн PRD'а; в редизайне все фильтры inline. Если вернём accessType/size/weekday — пересмотрим |

## Когда брать

- Жалоба пользователя на отсутствие конкретного фильтра — берём именно его, не пакетом
- Если решим возвращать accessType/size — добавлять как дополнительные pills в meta-row, шаблон уже есть (`<PriceFilter>` / `<CityPicker>`)
- Геопозиция — отдельная задача (нужен backend endpoint + UX-решение «отказ от geo»)
