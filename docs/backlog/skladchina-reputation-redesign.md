## Модель (таблица сценариев)

Действует только при `affects_reputation = true` («важный сбор»). Эмиссия — в момент
закрытия (`closeInternal`), `occurred_at = closed_at` (инвариант reputation-v2.md:86-90 сохранён).

| Исход участника | kind | Очки | Обоснование относительно событий |
|---|---|---|---|
| Оплатил (`paid`) | `skladchina_paid` | **+10** (без изменений) | 1/10 от ironclad (+100): явку верифицирует орг, оплата — самодекларация. Плюс символический до верификации (P2) |
| Отказался (`declined`), в любой момент до закрытия | — **ledger-строки НЕТ** (kind `skladchina_declined` больше не эмитится) | 0 | Отказ — желаемое поведение («не можешь — скажи сразу») и бесплатный выход из карательного сбора. Без строки, а не 0-строкой: иначе три отказа в один тап = бесплатный выход из «Новичка» (инфляция `outcome_count`). Статус остаётся в `skladchina_participants` — для будущей черты «доля отказов» (P1b) |
| Молчал, закрытие при `closed_at ≥ deadline` (`expired_no_response`) | `skladchina_expired` | **−40** (было −25) | 1/5 от no_show (−200): вред сопоставим (сорванная бронь), но обязательство навязано оргом — участник не нажимал «подтверждаю», как на этапе-2 событий. Break-even ≈ 80% оплат — чуть выше success-метрики «≥70% платят в срок» |
| Молчал, закрытие при `closed_at < deadline` (досрочно: цель / все ответили / ручное) | — строки нет; новый статус участника **`released`** | 0 | **Решение F5-02.** Обещание — «ответить к дедлайну»; дедлайн не наступил → нарушения нет. Возвращает спеку к глоссарию («expired = deadline истёк»), снимает противоречие skladchina.md :424 vs :205-210 |
| `markPaid` после дедлайна, до тика шедулера (~10 мин окно) | `skladchina_paid` | +10 | Сбор ещё active → оплата валидна, деньги важнее формального дедлайна. Поведение не меняем, фиксируем как осознанное |
| Вышел из клуба при активном сборе | удаляется из сбора (текущее поведение) | 0 | Без изменений |
| `affects_reputation = false` | строк нет вообще | — | Без изменений |

Kind `skladchina_declined` остаётся в enum (исторические −5-строки на staging), но новых
строк не получает: `ReputationPolicy.financeKind(declined) → null`.

## Валидации и гейты

- **`declared == expected` в fixed-режимах** (`fixed_equal`/`fixed_individual`), иначе 400.
  Сервер сейчас проверяет только `> 0`; UI уже readOnly. Убивает усилитель F5-02
  (заявка ≥ goal → мгновенное закрытие → минусы всем pending) и «нарисованный» collected.
- **Voluntary: sanity-cap** `declared ≤ 10_000_000` копеек (100 000 ₽) — гигиена статистики.
- **Тумблер «важный сбор»:**
  - дефолт **выкл** (Decision 4 остаётся);
  - **иммутабелен после создания** (вкл задним числом = ретроактивная мобилизация; ошибся — закрой досрочно: pending → `released` без минусов, paid получают +10);
  - доступен **только для fixed-режимов** («добровольный со штрафом за молчание» — оксюморон); для voluntary чекбокс задизейблен с подписью «Добровольный сбор не влияет на репутацию»;
  - **дедлайн ≥ 24 ч** (анти-«сбор-засада»; 48 ч отклонено — ломает юзкейс «бронь на завтра»);
  - **rate-limit: ≤3 реп-сбора на клуб за скользящие 7 дней** — валидация на создании, чекбокс дизейблится с объяснением. Единственный реальный анти-фарм И анти-грифинг механизм: потолок фарма +30/нед/клуб на друга (меньше одной честной явки +100), потолок грифинга −120/нед при двух игнорируемых DM.

## Уведомления — launch-blocker для −40

Сейчас напоминаний по складчине нет вообще. Штраф −40 легитимен, только если система
дважды предупредила и отчиталась:

1. DM при создании реп-сбора — с прайсом: «оплата +10, отказ — без штрафа, молчание до дедлайна −40»;
2. **reminder-DM pending-участникам за 24 ч до дедлайна** (новый шедулер, паттерн `EventReminderScheduler`);
3. DM при фактическом списании −40.

Без п.2-3 поднимать штраф с −25 нельзя — деплоить одним PR с ними.

## Анти-абьюз: принятые остаточные риски P1

- **Самодекларация +10 остаётся** до P2: подтверждение оргом отклонено для P1 — (а) фатально взаимодействует с авто-закрытием (сбор закрывается синхронно последним ответом → окно подтверждения схлопывается в ноль, честные плательщики получали бы 0), (б) против главной петли «орг+друзья» бесполезно (подтверждает сам сговорившийся орг), (в) 8-10 кликов трения на честный еженедельный сбор. В P2 — только с **отложенной эмиссией** (окно подтверждения после закрытия).
- **Реципрокный кросс-клубный фарм** (A владеет клубом α и кредитует B, B владеет β и кредитует A — rule 1 не ловит) — принят на capped-скорости ≤ +30/нед/клуб.
- **Серийный отказник невидим** (отказ = 0 без строки, вне Trust). Данные в `skladchina_participants` сохраняются — кандидат на черту профиля «доля отказов» в P1b (информация, не штраф).
- Гейты «возраст клуба ≥14 дней» и «≥3 участников» — отклонены (бьют по честным новым/малым группам); вырожденные сборы душатся rate-limit'ом.

## Forward-compat с Trust P1b (`reputation-v2-redesign.md` §5)

Trust считает исходы, не очки — модель ложится 1:1:

| kind / исход | Финансовая под-ось Trust |
|---|---|
| `skladchina_paid` | kept (числитель + знаменатель) |
| `skladchina_expired` | broken, вес ×2 в знаменателе |
| отказ (строки нет) | вне формулы |
| `released` (строки нет) | вне формулы |

**Зафиксированные deviations от redesign §5:** (1) бонус +15 за voluntary-оплату — вычеркнут
(фарм-канал: плюс без обязательств и риска); (2) орг-confirm — остаётся P2, но с требованием
отложенной эмиссии (см. выше); (3) «честный отказ» в P1 реализован как «нет строки», а не
«строка вне знаменателя» — для Trust эквивалентно, для `outcome_count` честнее.
