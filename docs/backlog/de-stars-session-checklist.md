# Сквозной чек-лист: всё, что собрано в этой сессии (de-Stars / платный клуб / member-admin)

> Единый прогон для PO на **staging** (`https://staging.77-42-23-177.sslip.io`), открывать через **@clubs_admin_bot**.
> Покрывает: награды (S2), обязательные СБП-реквизиты, оплату участником (карточка доверия + СБП/наличные + claim), решение организатора (подтвердить / отказать B+C), обзор участника (ответ на заявку), регрессию.
> Легенда: 🧑‍💼 организатор · 👤 участник · 👤2 второй участник. Детали по флоу — `payment-flow-test-cases.md`; награды/заметка — `de-stars-member-admin-test-cases.md`.

## 0. Подготовка
- 🧑‍💼 **PAID-open** клуб (цена > 0) с заданными реквизитами СБП; ≥1 frozen-участник.
- 🧑‍💼 **PAID-closed** (по заявке, цена > 0) — для «ответа на заявку».
- 🧑‍💼 **FREE** клуб — регрессия.
- 🧑‍💼 PAID **без** реквизитов — для cash-only.
- 👤 + 👤2 — участники.

## A. Награды клуба (S2)
- **A1** 🧑‍💼 Карточка участника → ✎ Редактировать → секция «Награды клуба» + «＋ Добавить награду».
- **A2** 🧑‍💼 ＋ → выбрать эмодзи (пикер ~12) + название (≤40) → зелёная «→»/Enter. **Ожид:** чип появился сразу, карточка не закрылась.
- **A3** 👤2 Открыть карточку этого участника. **Ожид:** чип награды виден (R3), без ✎ и ×.
- **A4** 🧑‍💼 Та же награда повторно → ошибка «Такая награда уже выдана» (дубль по названию).
- **A5** 🧑‍💼 6 наград → кнопка «＋» исчезает, «Максимум 6 наград»; 7-ю нельзя.
- **A6** 🧑‍💼 ✎ → «×» на чипе → снять → исчезает у всех.
- **A7** 🧑‍💼 Автоподсказки: выдал «Активист» одному → у другого ✎ → «＋» → ввод «Акт» → подсказка «Активист»; тап выдаёт сразу.
- **A8** 🧑‍💼 **FREE-клуб**: у карточки участника есть ✎ → награды + заметка (без подписки/«своя дата»). Выдать награду — работает.
- **A9** 🧑‍💼 R4: выдать/снять награду → кольца репутации/уровень участника НЕ меняются (косметика).
- **A10** 🧑‍💼 По себе (орг) — блока управления/наград нет.

## B. СБП-реквизиты обязательны для платного клуба
- **B1** 🧑‍💼 Создание клуба, шаг «Участники»: цена 0 → реквизитов нет; цена > 0 → блок «Реквизиты для взноса (СБП)» с обязательным полем (★) + опц. подсказкой.
- **B2** 🧑‍💼 Цена > 0, поле пустое → «Далее». **Ожид:** не пускает, «Для платного клуба укажите реквизиты для взноса». Со ссылкой — проходит.
- **B3** 🧑‍💼 Настройки: FREE → платный → появляются поля; пустую ссылку не сохранить.
- **B4** 🧑‍💼 Платный: стереть ссылку → Сохранить → не даёт. *(Legacy-клуб без реквизитов попросит добавить при 1-м сохранении.)*
- **B5** 🧑‍💼 Новый платный клуб с реквизитами → у frozen-участника сразу есть «Оплатить взнос».

## C. Оплата участником — карточка доверия
- **C1** 👤 frozen-участник → «Оплатить взнос» → sheet с суммой (💳 N ₽) + блок **«Кому вы переводите»**.
- **C2** 👤 Опытный орг: аватар + имя + **@username** + «Написать»; факты «на платформе с …», «Ведёт N клубов» (если N≥2), «доверяют N участников» (если N≥5).
- **C3** 👤 Тап «@…»/«Написать» → открывается чат с организатором в Telegram.
- **C4** 👤 Клуб свежего орга: карточка схлопывается — идентичность + «На платформе недавно» + подсказка «напишите перед первым переводом». **Нулей нет** (нет «Ведёт 1», «доверяют 0»).
- **C5** 👤 Орг без @username — имя+аватар есть, кнопки «Написать» нет.

## D. Оплата участником — методы (claim)
- **D1** 👤 Реквизиты заданы → переключатель «По СБП»/«Наличными». Нет реквизитов → только «Наличными».
- **D2** 👤 СБП: «Оплатить по СБП» открывает ссылку орга → «Прикрепить скриншот» (jpg/png ≤5 МБ). «Подтвердить оплату» неактивна без скриншота.
- **D3** 👤 Не-картинка / >5 МБ → ошибка, не принимает.
- **D4** 👤 Прикрепить → превью → «Подтвердить оплату». **Ожид:** статус **«Оплата на проверке · по СБП»**, кнопки «Оплатить взнос» нет. Переоткрыть клуб — статус сохранён.
- **D5** 👤 Наличные: чекбокс «Подтверждаю, что передал…» → «Подтвердить». **Ожид:** «Оплата на проверке · наличными» (без скриншота).

## E. Организатор — обзор участника и решение
- **E1** 🧑‍💼 PAID-closed: в карточке участника вверху орг-секции **«Ответ на заявку»** (анкета). У FREE-open / без вопроса — нет. У бейджа подписки **нет** «Только орг».
- **E1a** 👤2 На карточке другого «Ответ на заявку» НЕ виден (org-only).
- **E2** 🧑‍💼 После СБП-заявки → карточка: **«⏳ Оплата заявлена · СБП»** + **кликабельный скриншот** (открывается в полном размере). Наличные → «· наличные» + «скриншота нет».
- **E3** 🧑‍💼 «Мои клубы» → «Ждут оплаты»: у заявившего бейдж **«Оплата заявлена»** (зелёный) вместо «Ждёт оплаты».
- **E4** 🧑‍💼 **«Взнос получен»** → доступ открыт (active), участник ушёл из «Ждут оплаты», блок заявки/скриншот пропал; у участника статус «на проверке» снят, виден контент.
- **E5** 🧑‍💼 **«Отказать · вернуть перевод»** (только frozen) → подтверждение «Убрать N из клуба? Перевод вернёте сами…» → «Отказать и вернуть». **Ожид:** участник убран (cancelled), исчез из «Ждут оплаты», тост «Вступление N отклонено».
- **E5a** 👤 Отклонённому DM «организатор отклонил вступление и вернёт перевод»; клуб снова показывает его как не-участника (можно вступить заново).
- **E6** 🧑‍💼 По себе (орг) — управления нет; у активного участника — «Закрыть доступ» (а не «Отказать»).

## F. Заметка + своя дата (S1, регрессия после переделок)
- **F1** 🧑‍💼 PAID: ✎ → «Своя дата» + «Заметка» + Сохранить → дата меняет «Подписка до», заметка read-only у орга, 👤 её не видит.
- **F2** 🧑‍💼 Подписочная полоса + «Взнос получен»/«Закрыть доступ» на месте (не сломаны наградами/claim/reject).

## G. Безопасность / edge
- **G1** 👤 Активный участник (доступ есть) — платёжного UI/статуса нет.
- **G2** 👤 Не-участник / заявитель на рассмотрении — «Оплатить взнос» не видит.
- **G3** 👤2 На карточке другого: claim/скриншот/«Оплата заявлена» НЕ видит (только орг).
- **G4** (security) Подделанный proofUrl (внешний/`javascript:`) — бэк отклоняет (только наш /uploads/). 409-гонки (двойное «Взнос получен») — без ошибки.
- **G5** (IDOR) Участник не может заявить оплату/отклонить за другого; только своя membership.

## H. Регрессия (не сломалось смежное)
- **H1** 👤 FREE-open → «Вступить» → сразу участник, контент доступен, НЕ frozen, платёжного UI нет.
- **H2** Везде цены в ₽; слова «Stars»/⭐ нет.
- **H3** Выход из клуба, заявки (одобрить/отклонить free-closed), создание события/складчины, отметка посещаемости — без регрессий.
