# Тест-кейсы: полный платёжный флоу платного клуба (de-Stars)

> **Что покрываем:** весь жизненный цикл взноса платного клуба — реквизиты организатора → вступление участника (frozen) → заявка об оплате (СБП со скриншотом / наличные) → проверка организатором → открытие доступа → истечение/повтор. Honor-system: деньги идут участник→организатор **мимо платформы**, доступ всегда открывает организатор.
>
> **Где:** staging `https://staging.77-42-23-177.sslip.io`, открывать через **@clubs_admin_bot** (нужен живой initData; haptic/назад/deep-link проверяем вживую).
> **Легенда:** 🧑‍💼 организатор · 👤 участник · 👤2 второй участник.
> **Связанные доки:** общий чек-лист ветки — `de-stars-member-admin-test-cases.md` (награды/заметка/роли); этот файл — только платёжный флоу.

## 0. Предусловия (подготовка данных)

| # | Что подготовить |
|---|---|
| P1 | 🧑‍💼 Аккаунт-организатор. |
| P2 | 👤 + 👤2 — два аккаунта-участника (для проверки публичности/изоляции). |
| P3 | **PAID-open** клуб (цена > 0, открытый), реквизиты СБП заданы. |
| P4 | **PAID-closed** клуб (по заявке) — для проверки вступления через одобрение. |
| P5 | **FREE** клуб (цена 0) — для регрессии (платёжного UI быть не должно). |
| P6 | (опц.) PAID-клуб **без** реквизитов — для проверки cash-only. |

## 1. Реквизиты организатора (обязательны для платного клуба)

> Инвариант: `цена > 0 ⇒ реквизиты заданы`. Без реквизитов у участника нет способа оплатить (кроме наличных).

- **1.1** 🧑‍💼 «+» → создать клуб → шаг «Участники»: цена `0` → **нет** блока реквизитов.
- **1.2** 🧑‍💼 На том же шаге поставить цену `> 0` → появляется блок **«Реквизиты для взноса (СБП)»** с обязательным полем (★) + опц. подсказкой.
- **1.3** 🧑‍💼 Цена `> 0`, поле реквизитов пустое → «Далее». **Ожид:** блокирует, ошибка «Для платного клуба укажите реквизиты для взноса». Вписать ссылку/номер → «Далее» проходит, клуб создаётся.
- **1.4** 🧑‍💼 Настройки клуба → есть секция «Реквизиты для взноса (СБП)» (показывается при цене > 0). Изменить ссылку + подсказку → Сохранить → успех.
- **1.5** 🧑‍💼 Настройки: сменить FREE → платный (цена > 0). **Ожид:** появляются поля реквизитов; сохранить с пустой ссылкой нельзя (та же ошибка); со ссылкой — сохраняется.
- **1.6** 🧑‍💼 Платный клуб: в настройках стереть ссылку → Сохранить. **Ожид:** не даёт (платный обязан иметь реквизиты).
- **1.7** 🧑‍💼 Legacy: платный клуб, созданный до фичи (реквизитов нет) → открыть настройки → при первом сохранении любого поля попросит добавить ссылку. *(Ожидаемо.)*
- **1.8** 🧑‍💼 FREE-клуб → в настройках секции реквизитов **нет**.

## 2. Вступление участника в платный клуб → `frozen`

- **2.1** 👤 PAID-open клуб → «Вступить». **Ожид:** «Вы вступили в клуб · доступ откроет организатор после того, как передадите взнос»; контента/табов нет; цена в ₽.
- **2.2** 👤 PAID-closed клуб → «Хочу вступить» → заявка. 🧑‍💼 одобряет на «Мои клубы». **Ожид:** участник становится `frozen` (в «Ждут оплаты»), НЕ «ожидание Stars».
- **2.3** 👤 Invite-ссылка в PAID-клуб → «Вступить». **Ожид:** `frozen`, тот же экран.
- **2.4** 👤 На экране frozen есть кнопка **«Оплатить взнос»** + текст «Оплата идёт напрямую организатору».

---

## 3. Заявка об оплате — СБП (со скриншотом)

> Требует заданных реквизитов СБП у клуба (блок 1).

- **3.1** 👤 frozen-участник → «Оплатить взнос» → открывается sheet «Оплата взноса» с суммой (**💳 N ₽ / мес**).
- **3.2** 👤 Реквизиты заданы → в sheet есть переключатель **«По СБП» / «Наличными»**, по умолчанию «По СБП».
- **3.3** 👤 Шаг 1 «Оплатить по СБП»: тап по кнопке → открывается ссылка/приложение организатора (+ подсказка организатора, если задана).
- **3.4** 👤 Шаг 2 «Прикрепить скриншот»: «Подтвердить оплату» **неактивна**, пока скриншот не прикреплён.
- **3.5** 👤 Прикрепить картинку (jpg/png ≤5 МБ) → показывается превью + «Заменить скриншот»; «Подтвердить оплату» становится активной.
- **3.6** 👤 Прикрепить не-картинку / файл > 5 МБ. **Ожид:** ошибка («Только JPEG или PNG» / «Файл больше 5 МБ»), скриншот не принят.
- **3.7** 👤 «Подтвердить оплату». **Ожид:** sheet закрывается, на экране клуба статус **«Оплата на проверке · по СБП»** (кнопки «Оплатить взнос» больше нет). Haptic успеха.
- **3.8** 👤 Закрыть и переоткрыть клуб. **Ожид:** статус «Оплата на проверке» сохраняется (не сбросился).

---

## 4. Заявка об оплате — наличные (без скриншота)

- **4.1** 👤 frozen-участник → «Оплатить взнос» → выбрать **«Наличными»**. **Ожид:** текст «Передайте N ₽ организатору наличными» + чекбокс «Подтверждаю, что передал(а) взнос наличными»; скриншот не запрашивается.
- **4.2** 👤 «Подтвердить оплату» **неактивна**, пока чекбокс не отмечен; отметить → активна → подтвердить. **Ожид:** статус **«Оплата на проверке · наличными»**.
- **4.3** 👤 Клуб **без** реквизитов СБП (P6): открыть sheet. **Ожид:** переключателя методов нет, доступны только наличные.

---

## 4b. Карточка доверия организатора на экране оплаты

> На sheet'е оплаты — блок «Кому вы переводите» (акцент на аккаунте орга, метрик клуба нет). Факты условные, без нулей.

- **4b.1** 👤 Открыть «Оплатить взнос» у клуба **опытного** орга. **Ожид:** карточка «Кому вы переводите»: аватар + имя + **@username** + кнопка **«Написать»**; ниже факты: «на платформе с {дата}», «Ведёт N клубов» (если N≥2), «Его клубам доверяют N участников» (если N≥5).
- **4b.2** 👤 Тап по @username / «Написать». **Ожид:** открывается чат с организатором в Telegram (`t.me/{username}`).
- **4b.3** 👤 Клуб **свежего** орга (нет клубов/участников, недавно зарегистрирован). **Ожид:** карточка схлопывается — только идентичность + «На платформе недавно» + подсказка **«Организатор новый — напишите перед первым переводом»**. **Нулей/пустых метрик нет** (нет «Ведёт 1 клуб», нет «доверяют 0»).
- **4b.4** 👤 У орга без @username (редко). **Ожид:** имя+аватар есть, кнопки «Написать»/@ нет (нельзя написать) — без ошибок.
- **4b.5** (порог) Орг с 1 клубом и <5 активными. **Ожид:** строки «Ведёт N» и «доверяют N» скрыты (показываются только при ≥2 клубов / ≥5 участников).

## 5. Организатор видит и проверяет заявку

> Заявка появляется у организатора **только после полной отправки формы** участником.

- **5.0** 🧑‍💼 Карточка участника закрытого клуба: вверху орг-секции — блок **«Ответ на заявку»** (что участник написал в анкете при вступлении). У открытого клуба / без вопроса — блока нет. У бейджа подписки **нет** пометки «Только орг» (убрана как мусор).
- **5.0a** 👤 Обычный участник на карточке другого: «Ответ на заявку» НЕ виден (только организатору).
- **5.1** 🧑‍💼 До отправки заявки участником: в карточке участника блока заявки нет (просто frozen + «Взнос получен»).
- **5.2** 🧑‍💼 После СБП-заявки участника → открыть карточку участника. **Ожид:** блок **«⏳ Оплата заявлена · СБП»** + **кликабельный скриншот** (тап открывает в полном размере).
- **5.3** 🧑‍💼 После cash-заявки → карточка: «⏳ Оплата заявлена · наличные» + пометка «скриншота нет (наличные)».
- **5.4** 🧑‍💼 «Мои клубы» → блок «Ждут оплаты»: у заявившего участника бейдж **«Оплата заявлена»** (зелёный) вместо «Ждёт оплаты».
- **5.5** 🧑‍💼 Red-dot на «Управление»/«Участники» горит, пока есть frozen (в т.ч. заявившие).
- **5.6** 🧑‍💼 «Взнос получен» у заявившего. **Ожид:** доступ открыт (участник active), участник уходит из «Ждут оплаты», блок заявки/скриншот пропадает, статус «на проверке» у участника снят.
- **5.7** 👤 После 5.6 участник видит контент/табы клуба; кнопки/статуса оплаты на экране нет.

## 6. Жизненный цикл / повторная оплата

- **6.1** 🧑‍💼 «Закрыть доступ» активному участнику → `frozen`. **Ожид:** у участника снова появляется «Оплатить взнос» (заявку можно подать заново).
- **6.2** 👤 Подать заявку повторно после повторной заморозки → работает (новая заявка перетирает прежнюю).
- **6.3** (backend/edge) Истечение доступа: у active-участника `subscription_expires_at` в прошлом → шедулер роняет в `frozen` → участник снова может заявить оплату. *(Требует прохода времени/правки даты в БД — проверяется не вручную.)*
- **6.4** 🧑‍💼 «Взнос получен» продлевает доступ на 30 дней от max(now, текущая дата) — ранняя оплата не теряет дни.

## 7. Негативные / security / edge

- **7.1** 👤 Активный участник (доступ открыт) — на экране клуба платёжного UI/статуса нет (заявить нельзя — уже есть доступ).
- **7.2** 👤 Не-участник / заявитель на рассмотрении (closed-клуб, ещё не одобрен) — кнопки «Оплатить взнос» не видит.
- **7.3** 👤 Обычный участник открывает карточку **другого** участника. **Ожид:** НЕ видит ни блок заявки, ни скриншот, ни «Оплата заявлена» (claim — только организатору).
- **7.4** 👤 СБП-заявка без скриншота отклоняется на бэке (кнопка и так заблокирована, но контракт: 400 «Прикрепите скриншот оплаты»).
- **7.5** (security) Подделанный `proofUrl` (внешний хост / `javascript:`) — бэк отклоняет (принимается только наш upload-URL). У организатора в ссылке не может оказаться чужой/опасный URL.
- **7.6** (security/IDOR) Участник не может заявить оплату за **другого** участника (эндпоинт работает только со своей membership) и не может заявить в клубе, где не состоит.
- **7.7** (race) Двойное «Взнос получен» из двух сессий организатора / «Взнос получен» одновременно с заявкой — обрабатывается без ошибки (silent-recovery / 409 «обновите экран»).
- **7.8** 👤 Только `frozen` может заявить оплату; для active/не-члена бэк отвечает ошибкой.

## 8. ₽ вместо Stars (регрессия монетизации)

- **8.1** Каталог/страница/invite PAID-клуба: цена «N ₽ / мес». Нигде нет слова «Stars» и значка ⭐.
- **8.2** Создание клуба, шаг «Участники»: лейбл «Цена подписки (₽/мес)»; при цене > 0 подсказка про прямую оплату без комиссии платформы.
- **8.3** 🧑‍💼 Финансы клуба: «Активных участников» + пояснение про прямую оплату; нет «Выручка/Доля/Комиссия», нет Stars.

## 9. Регрессия (бесплатный клуб не затронут)

- **9.1** 👤 FREE-open клуб → «Вступить». **Ожид:** сразу полноправный участник, контент доступен, НЕ frozen, платёжного UI нет.
- **9.2** 🧑‍💼 FREE-клуб → «Участники»: корзин «Ждут оплаты»/«Скоро закончится» нет.
- **9.3** Выход из клуба, заявки (одобрить/отклонить), создание события/складчины — без регрессий.
