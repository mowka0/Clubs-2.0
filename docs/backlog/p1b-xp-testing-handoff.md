## 3. Логика (для сверки математики)

### XP
```
XP = ironclad·10 + spontaneous·8 + skladchina_paid·3 + distinctKeptClubs·20
```
- kept-kinds дают очки; **broke/neutral = 0** (XP не падает) — `XpPolicy.kindXp`.
- `distinctKeptClubs` = число РАЗНЫХ клубов, где есть ≥1 kept-исход (бонус разнообразия +20/клуб).
- **Ключ:** строка в `reputation_ledger` появляется ТОЛЬКО когда событие **завершилось+отмечена явка**
  (→ ironclad/spontaneous) или складчина **закрылась с оплатой** (→ skladchina_paid). **Вступление в клуб
  строк НЕ создаёт** → XP/diversity не двигает.

### Уровни (10)
`порог(i) = round(40·i^1.85)`, i=0..9: **0, 40, 144, 305, 520, 786, 1101, 1464, 1874, 2330**
(Гость / Свой / Участник / Завсегдатай / Активист / Энтузиаст / Душа компании / Столп сообщества /
Легенда / Амбассадор). `levelIndexFor(xp)` = самый высокий перешагнутый порог.

### Бейджи (9, калибруются — PO может менять имена/пороги)
| Семья | Бейдж | Условие | Чувствителен к «Новичок» (≥3 в клубе)? |
|---|---|---|---|
| Участие | Первый шаг | `totalKept ≥ 1` | нет |
| Участие | Железобетон | `ironclad ≥ 10` | нет |
| Участие | Лёгок на подъём | `spontaneous ≥ 5` | нет |
| Участие | Надёжный плательщик | `skladchina_paid ≥ 3` | нет |
| Разнообразие | Разносторонний | `distinctKeptClubs ≥ 3` | нет |
| Разнообразие | Вездесущий | `distinctKeptClubs ≥ 5` | нет |
| Доверие | Надёжный | `reliableClubs ≥ 1` | **да** |
| Доверие | Кремень | `maxTrustWithRecord ≥ 90` | **да** |
| Доверие | Столп доверия | `reliableClubs ≥ 3` | **да** |

**Гейт ≥3-в-клубе (`MIN_OUTCOMES_FOR_DISPLAY`) применяется ТОЛЬКО к доверие-бейджам** (вход
`reliableClubs`/`maxTrustWithRecord` — клуб учитывается, лишь если в нём ≥3 исхода и Trust≥70/≥90).
XP, уровни, участие- и разнообразие-бейджи считаются с **первого** исхода, статус «Новичок» им не мешает.
В коде: гейт стоит вокруг блока `if (rows.size >= MIN_OUTCOMES_FOR_DISPLAY)` в `XpService.computeStats`.

### Анти-фарм (наследуется бесплатно)
1. Владелец не имеет ledger-строк в своём клубе (rule 1) → XP там не копит, только участием в чужих.
2. Бонус разнообразия — за РАЗНЫЕ клубы (широта), не за долбёжку в одном.

### Трассируемый пример (из integration-теста)
3 ironclad в клубе A + 1 skladchina_paid в клубе B →
`XP = 3·10 + 1·3 + 2·20 = 73` → уровень «Свой» (40≤73<144), прогресс 33/104.
Бейджи: Первый шаг; клуб A (3 исхода, Trust 92) → Надёжный + Кремень; клуб B (1 исход <3) не идёт в
доверие; Разносторонний нет (2 клуба). Итог: **Первый шаг, Надёжный, Кремень**.

## 4. Что проверить на staging (`https://staging.77-42-23-177.sslip.io`)

Тестируем **математику и логику** XP/бейджей (это и есть фокус новой сессии):

1. **Панель «Уровень» на профиле** — название уровня, XP-бар с прогрессом «до следующего», бейджи.
   Видна на аккаунте с ≥1 kept-исходом; на 0 XP скрыта (норма).
2. **XP-арифметика** — посчитай руками ожидаемый XP по формуле для аккаунта с известной историей,
   сверь с панелью. (Можно через БД-туннель вытащить строки `reputation_ledger` — см. раздел 6.)
3. **Уровень** — пороги 0/40/144/… Сверь название + прогресс-бар.
4. **Бейджи по каждому порогу** — особенно:
   - **Разносторонний** требует ≥1 kept в 3 РАЗНЫХ клубах (не вступление, а завершённую активность).
   - Доверие-бейджи требуют клуб с ≥3 исходами.
5. **XP не падает** — прогул (no_show) не уменьшает XP/уровень.
6. **Анти-фарм** — в своём клубе (где ты владелец) XP не начисляется.

> Если значения расходятся с ожиданием — сверить факты через БД-туннель ДО вывода багов.

## 6. Решённые вопросы (НЕ переоткрывать)

- **Organizer-XP ВЫКИНУТ** (2026-06-14, PO): `+15` событие / `+8` складчина убраны. «Организаторский
  результат» = сигнал о МЕСТЕ, живёт в клуб-треке (ось «Надёжность организатора»), не в личной
  геймификации (иначе тройной счёт явки). XP = участие-only.
- Веса +10/+8/+3, diversity +20, формула уровней `40·n^1.85`, стрик отложен — залочено в §H3.
- Бейджи — старт `XpPolicy.BADGES`, **калибруются** (PO может менять имена/пороги при тесте).
- Вступление в клуб ≠ kept-исход (ledger пишется только на завершённом событии/закрытой складчине).
- Гейт ≥3-в-клубе — только для доверие-бейджей, не для XP/участие/разнообразие.
