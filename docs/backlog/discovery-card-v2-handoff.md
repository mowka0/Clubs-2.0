## 2. Задача A — Карточка клуба v2

Файлы: `frontend/src/components/ClubCard.tsx`, `frontend/src/styles/redesign.css`.
`ClubCard` используется только в `DiscoveryPage` (проверено). `MyClubsPage` имеет свои
`MyClubCard`/`HistoryClubCard` — их НЕ трогаем.

### Что меняется в разметке
- **Обложка** (`rd-cover`): убрать скрим-градиент `rd-cover::after` (PO: «градиент
  мешает»). Цена (`rd-price-chip`, top-right) и бейдж встречи (`rd-date-badge`,
  top-left, при featured) — остаются, у них своя подложка, скрим им не нужен.
- **Полка метрик** на обложке (НОВОЕ): ряд из трёх метрик, прижат в нижний-левый угол,
  панель из материала карточки, скруглён только правый верхний угол. Показывать
  **только когда `facts` загружены** (как сейчас трио — до фактов полки нет).
  Порядок: возраст · участники · вовлечённость.
  - возраст: иконка-часы + `{ageDays} дн`  ← короткое «дн», НЕ склоняем «день/дня/дней»
  - участники: иконка-люди + `{memberCount}`  ← голое число, слово несёт иконка
  - вовлечённость: иконка-молния + `{engagementPercent}%`  ← цвет `--live` (зелёный)
- **Тело** (`rd-body`): `rankpill` (при `topInCategory`, над названием — как сейчас) +
  название + строка города с иконкой-пином. Трио `rd-mrow`/`rd-mstat` **удалить**
  (переехало на полку). `rd-meta` теперь всегда просто город (убрать ветку
  `facts ? city : city · members`).

### Ключевое решение по материалу полки
Полка лежит НА фото → должна быть **непрозрачной** и **совпадать с телом карточки**,
чтобы читалось «футер вырос вверх». Сейчас `.rd-club-card` = полупрозрачный `--glass-bg`
поверх `--bg`. Для бесшовности:
- задать `.rd-club-card` и `.rd-club-card .rd-shelf` **один сплошной токен** (напр.
  `--card-solid`: тёмная `#17171C`, светлая `#FFFFFF` — это ≈ как glass-bg выглядит
  поверх фона, вид карточки почти не изменится, но полка сможет перекрыть фото).
- НЕ использовать `--glass-bg` для полки — она прозрачная, фото просвечивало бы.

### Готовый CSS (из мокапа 11, адаптировать имена под `rd-`)
```css
/* полка метрик — уголок в нижнем-левом углу обложки */
.rd-club-card .rd-shelf {
  position: absolute; left: 0; bottom: 0; z-index: 2;
  display: flex; align-items: center; gap: 12px;
  padding: 9px 12px 4px 12px;      /* верх 9 — с воздухом (PO); низ 4 — придвинуто к кромке */
  background: var(--card-solid);   /* тот же материал, что тело карточки */
  border-top-right-radius: 13px;   /* тот самый скруглённый правый угол */
  box-shadow: inset 0 1px 0 var(--shelf-hl);  /* тонкий блик по верхней кромке */
}
.rd-club-card .rd-m {
  display: inline-flex; align-items: center; gap: 4px;
  font-size: 11px; font-weight: 700; color: var(--text); white-space: nowrap;
}
.rd-club-card .rd-m svg { width: 11px; height: 11px; flex-shrink: 0; color: var(--text-faint); }
.rd-club-card .rd-m.ok, .rd-club-card .rd-m.ok svg { color: var(--live); }
```
`--shelf-hl`: тёмная `rgba(255,255,255,0.12)`, светлая `rgba(255,255,255,0.9)`.
`rd-cover` держать `position: relative` (уже так) и без `overflow` проблем — полка
скругляется вместе с карточкой (`.rd-club-card { overflow: hidden }` уже есть).

## 4. Задача C — Полка «Встречаются сегодня»

Новый компонент `frontend/src/components/TodayShelf.tsx` (presentational + маленький
хук фильтрации). Вставляется в `DiscoveryPage` **между рядом чипов категорий и блоком
списка** (перед счётчиком/пустыми состояниями/карточками).

- **Вход:** `clubs: ClubListItemDto[]` — уже развёрнутый загруженный список
  (`data?.pages.flatMap(p => p.content)` — он уже есть в DiscoveryPage).
- **Фильтр:** `nearestEvent != null && isToday(nearestEvent.eventDatetime)` (та же
  локальная календарная дата). Сортировка по `eventDatetime` возрастанию.
  `isToday` — helper в `utils/formatters` (сравнить Y/M/D в локальной зоне).
- **Рендер:** горизонтальный ряд мини-карточек. Каждая: бейдж времени `HH:MM` +
  «сегодня», название (клап в 2 строки), `{категория} · {memberCount}`.
  Клик → `navigate('/clubs/{id}')` + `haptic.impact('light')`.
- **Скрывать целиком, когда пусто** (нет клубов с событием сегодня).
- Респектит фильтры автоматически (строится из уже отфильтрованных загруженных клубов).
- **CSS готов** — классы `today-shelf / today-rail / today-mini` в
  `mockups/_shared.css` (discovery-card) и в `04-final.html`. Перенести под `rd-`-имена.
- Категория → русский лейбл: в `ClubCard.tsx` есть `CATEGORY_LABELS`. Вынести в
  `shared`/`utils` и переиспользовать (не дублировать).

> **Известное ограничение (залогировать, не чинить):** полка видит только загруженные
> страницы выдачи (первая = 20 клубов). При сотнях клубов в городе увидит не все. Для
> MVP ок; настоящий разрез «события сегодня по городу» — потом, отдельным эндпоинтом.

## 7. Залоченные решения (не переоткрывать без PO)

1. Метрики — компактная полка-уголок из материала карточки, голый текст (не пилюли,
   не полная полка, не стеклянная лента). Прошли: сетку-коробки, стеклянную ленту,
   фрост-пилюли, полную полку, полку с загибом — **выбран уголок**.
2. Градиент обложки — убран.
3. Слова метрик: `{n} дн` (коротко), голое число участников, `{n}%` зелёным.
4. Адрес — только город (района в данных нет).
5. `rankpill` — в теле над названием (как в проде), НЕ на обложке (иначе конфликт с
   `rd-date-badge` в top-left).
6. Плитки статистики — убраны.
7. Полка «сегодня» — из `nearestEvent`, без бэка, видит только загруженные страницы.
