## 3. РАБОЧИЙ РЕЦЕПТ (главное — не потерять)

> **ОБНОВЛЕНО 2026-07-17.** Голый style_id провалился: он переносит только манеру отрисовки
> (обводка/палитра/глянец), а анатомию и одежду модель выдумывает заново — в сценах выходил
> пухлый лисёнок без пиджака. PO забраковал. Рабочая связка теперь ТРИ элемента:
> **image-to-image от канона + «паспорт героя» в промпте + SVG-хирургия для мелочей.**

Генерация: **не через MCP-инструменты, а напрямую в REST** через `docs/design/empty-states/recraft-gen.py`.
Ключ Recraft читается скриптом из `~/.claude.json` (MCP-сервер `recraft` уже добавлен, `-s local`).

```bash
# Сцена с нашим лисом (рецепт v2): i2i от канона + паспорт
python3 docs/design/empty-states/recraft-gen.py <имя> "<действие сцены>" \
  --hero --model recraftv3 \
  --style-id 252d8eea-6e46-4c62-b25e-f4b2d5cc2f18 \
  --i2i "docs/design/empty-states/ref/hero/hero-01.png" \
  --strength 0.6 --n 2
```

Правила рецепта v2 (проверены на 9 сюжетах):
- **`--i2i` от `ref/hero/hero-01.png` — обязательно.** Без него (чистый t2i) морда уезжает
  в чужого лисёнка даже с паспортом. strength: 0.5 — поза почти канона, 0.55–0.65 — новые позы,
  выше 0.7 — начинает терять одежду.
- **`--hero` вшивает «паспорт героя»** (константа `PASSPORT` в скрипте): взрослые пропорции,
  скулы, очки, пиджак. Действие сцены — **в начале** промпта, паспорт сам встаёт хвостом
  (из хвоста промпта модель теряла позу).
- **`--outfit "wearing ..."`** — переодеть лиса под сюжет (тренч сыщика, свитер). Одежду
  продублировать и в начале промпта сцены, иначе теряется.
- **`--no-glasses`** — для сцен, где нужны глаза. НО: с очками на морде канона i2i очки
  НЕ снимает ни negative, ни strength 0.8 (see §4) → глаза делаются **SVG-хирургией**.
- **Мелкие пропсы (штекер, билет, значки, свисток) модель стабильно теряет.** Крупные
  (лупа, гирлянда, карандаш) — рендерит. Мелочь дорисовываем хирургией.

### SVG-хирургия (бесплатно, детерминированно)
Сгенерированный SVG — плоские пути с 6 филлами палитры; линзы/нос/буквы легко найти по цвету
`rgb(155,68,52)`/`rgb(122,33,12)` + bbox. Проверенные операции (код в истории сессии 2026-07-17):
- **Закрытые глаза:** удалить 2 пути линз, добавить две дуги `Q`-кривыми цветом линии `rgb(213,97,28)`.
- **Очки-реквизит:** пересадить пути 26–33 из канона `stylyaga-utya-2.svg` через `<g transform>`
  (scale ~0.58) + дорисовать перемычку между линзами (в каноне она сидит в общем контурном пути).
- **График/билет/значки:** простые path/rect/circle в цветах бренда поверх, перед `</svg>`.
- **Прозрачный фон:** path 0 (белый прямоугольник на весь canvas) просто удалить — `remove_background` не нужен.

**style_id — три штуки, использовать последний:**
| style_id | что это | вердикт |
|---|---|---|
| `d45aa927-8b05-483b-9c41-7df9a89cbb7d` | из 5 референсов Ути (вкл. кадр с мускулами) | ❌ даёт пунктирную «строчку» |
| `a28de31a-3e10-46bc-950d-325b9268b33c` | из 3 чистых Уть | ✅ строчки нет, но композиция «по пояс» |
| `252d8eea-6e46-4c62-b25e-f4b2d5cc2f18` | **снят с НАШЕГО лиса** (hero-01.png) | ✅✅ **использовать это** |

**Почему style_id снят с героя — это ключевой трюк:** он даёт разом (1) консистентность персонажа между сценами, (2) отсутствие строчки, (3) **полный рост сам появился** — утиная композиция «бюст по пояс» перестала тянуть.

### Грабли Recraft API (проверено на своей шкуре)
- **MCP-обёртка непригодна** для тонкой работы: не пробрасывает `controls` (цвет/фон) и `negative_prompt`, заперта на `recraftv3/v2`. Только для `create_style` / `get_user` удобна.
- `negative_prompt` — **только v3/v2**, v4.1 падает с 400.
- `no_text` control — **только v3**; v2 и v4.1 Pro Vector падают.
- v4.1 Pro Vector: не берёт размер `1024x1024`, нужен формат соотношения — `"1:1"`.
- `response_format: url` → скачивание даёт **403** (подписанные ссылки). Использовать `b64_json`.
- **Лок цвета `controls.colors` обязателен** — без него модель игнорирует бренд и выдаёт свою палитру.
- Модели `*_pro_vector` **заметно дороже** — первые 5000 кредитов сожгли в основном на них.

### Просмотр результата
SVG не читается глазами напрямую: `qlmanage -t -s 600 -o . file.svg` → PNG, затем склейка контактного листа через Pillow (venv: `/private/tmp/.../scratchpad/fontenv/bin/python` — **умрёт с сессией, пересоздать**: `python3 -m venv fontenv && ./fontenv/bin/pip install pillow`).

## 4. ТУПИКИ — НЕ ПОВТОРЯТЬ

| Пробовали | Почему нет |
|---|---|
| **Капибара** | 4 попытки в 2 стилях → выходит **медведь или лошадь**. ИИ не знает породу. Закрыто. |
| **Енот в оранжевом** | Читается как **красная панда**: узнаваемость енота = серый + чёрная маска. |
| **Минималистичная линия** (первый заход) | Мимо: PO хотел детальное, а не иконку. |
| **3D `plastic` / `handmade_3d`** | Слишком объёмно (PO), и 3D-растр **не анимируется**. `handmade_3d` = фото пластилина. |
| **`bold_stroke`** | Впекает **фон-прямоугольник** в картинку + рисует мех/текстуру. |
| **`editorial` (лис-денди)** | Красиво, взрослo, но **надменный** — против души продукта («всем рады»). Отвергнут PO. |
| **Промптом добиться обводки** | 3 попытки — стена. Сабстиль навязывает эстетику поверх промпта. Только `create_style`. |
| **Очки поднять на лоб / снять** | Модель приклеивает к морде. При i2i от канона-в-очках не снимаются ВООБЩЕ (negative + strength 0.8 — бесполезно). Решение: SVG-хирургия (см. §3). |
| **Полный рост через промпт** (на утином стиле) | Стиль тащит композицию «по пояс». Решилось само на style_id от героя. |
| **Чистый t2i со style_id для сцен** (заход 2026-07-15) | Переносит стиль, но НЕ персонажа: пухлый лисёнок без пиджака. PO забраковал. Только i2i от канона. |
| **t2i без очков** (для глазастых сцен) | Без i2i-якоря морда уезжает в детсадовского лисёнка с глазами-пуговицами. Глаза — только хирургией. |
| **SVG-вывод для итераций** | ~80 кредитов/картинка (!): 42 картинки съели 3360. Для проб генерить растр, векторизовать только победителей. |

## 5. Анимация — исследовано, решение НЕ принято

**Главный вывод: инструмент не проблема, проблема — риггинг.**
Сгенерированный SVG = **278 безымянных путей вповалку**. Нет «руки», «глаза», «хвоста» как объектов → **анимировать по частям нельзя**. Ни Rive, ни Lottie, ни CSS этого не спасут — всем нужен послойный персонаж.

| | Rive | Lottie | CSS по слоям |
|---|---|---|---|
| Рантайм | ~200 КБ | ~60 КБ | **0 КБ** |
| Плавность | 60 fps | 17 fps | 60 fps |
| State machine | да | нет | нет |

Для пустых экранов нужны **idle-петли**, интерактив не нужен → Rive избыточен, **CSS по слоям бесплатен и идеален для Mini App**. Инструменты для сборки Lottie при желании: [Lottielab](https://www.lottielab.com/create/svg-animation), [SVGator](https://www.svgator.com/create-lottie-animation).

**Варианты (выбрать):**
1. Сцена = отдельная картинка + idle целиком (парение/дыхание) + анимированный реквизит, добавленный руками. Дёшево, до Ути не дотягивает.
2. **Перерисовать канон руками в послойный SVG** → полноценное движение (моргает, машет, хвост). Качество = уровень моей руки, не иллюстратора.
3. Иллюстратор/аниматор собирает марионетку в Rive/Lottielab.

**СПАЙК варианта 1 — сделан (2026-07-17): `mockups/animated-cafe.html`.** Лис из nano banana
(PNG, кафе) на экране «Клуб → Активности · участник»: пружинный вход, дыхание+микронаклон
(два слоя transform), пар над чашкой (SVG-петля), блик по очкам, своя дышащая тень, пылинки,
каскад текстов, shimmer скелета «скоро здесь», тап = squish+залп пара. Всё CSS, JS 6 строк,
prefers-reduced-motion учтён. Вывод: для пустых экранов «вариант 1» выглядит достаточно живым.

**PNG-пайплайн для артов из nano banana (Gemini):**
- Фон и запечённую тень вырезаем сами (Pillow): flood-fill белого от краёв (порог >216 —
  у Gemini мягкий край 3–4px), кеинг тени по холодному серому только в нижней зоне
  (~(193,185,196), правило «r<244 и |r−b|<14» не задевает чисто-белую рамку стикера),
  эрозия альфы MinFilter(5) против ореола + блюр альфы 1.0. Код — в истории сессии 2026-07-17.
- PNG8-квантование (FASTOCTREE, 256): 538 КБ → **71 КБ** (WebP дал 87 КБ — не выиграл).
- **Векторизация растра НЕ нужна** (Photopea «add raster graphics» — это обёртка PNG в SVG,
  не трассировка; платные трассировщики дают осколки без риггинга — анимации не помогают).
- Ассеты: `ref/nano/fox-cafe-src.png` (оригинал) → `fox-cafe.png` (вырезанный, 566×900).
