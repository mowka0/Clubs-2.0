- **S4-3 (item 2)** 👤 frozen-участник PAID-клуба, у организатора которого **НЕ указан Telegram-username** → «Оплатить взнос». **Ожид:** в sheet НЕ показывается переключатель «По СБП», только наличные + плашка «Оплата только наличными: организатор не указал Telegram… передавайте взнос лично — так безопаснее». Кнопки «Написать»/«@username» нет. Если орг свежий — nudge без «напишите ему».
- **S4-3b (item 2 контроль)** 👤 организатор С username + реквизитами → sheet показывает «По СБП»/«Наличными» как раньше (исключение не задело обычный кейс).
- **S4-4 (item 4)** 🧑‍💼 FREE-клуб → карточка участника. **Ожид:** поле «Заметка» открыто сразу (не за ✎), рядом «Удалить из клуба»; написать + «Сохранить заметку» → карточка не закрывается, заметка сохранена.
- **S4-5 (item 5)** 🧑‍💼 Запомнить «основу клуба» (кольцо Сплочённость на странице клуба) при участнике с ≥3 явками. Этот участник выходит/его кикают. **Ожид:** «основа клуба» уменьшается на 1 (ушедший больше не в ядре). Контроль: заморозка участника (frozen) ядро НЕ уменьшает.
- **S4-6 (DM при заморозке)** 🧑‍💼 У активного платного участника «Закрыть доступ». 👤 **Ожид:** участнику приходит DM от бота «🔒 Организатор … закрыл вам доступ. … оплатите взнос» с inline-кнопкой **«Оплатить взнос»** → открывает страницу клуба в Mini App, там кнопка «Оплатить взнос».
- **S4-7 (блок у участника)** 👤 Будучи замороженным в клубе → открыть своё «Мои клубы». **Ожид:** вверху блок **«🔒 Доступ закрыт — оплатите»** со строкой клуба + бейдж «Нужно оплатить» (или «Оплата на проверке», если уже заявил оплату); этого клуба НЕТ в «Где я состою». Тап по строке → страница клуба. После «Взнос получен» оргом клуб уходит из блока обратно в «Где я состою».

## S5. UI-правки сессии 5 (PO-ретест)
- **S5-1 (item 1)** 🧑‍💼 Карточка участника → панель «Управление участником». **Ожид:** поле «Заметка» (и в режиме ✎ поле «Своя дата») отстоит от соседних блоков как обычные поля формы — без увеличенного зазора.
- **S5-2 (item 2)** 🧑‍💼 Страница клуба → таб **«Участники»**. **Ожид:** блоков «⏳ Скоро закончится» / «💸 Оплата вступления» НЕТ — только плоский список участников. Эти блоки есть ТОЛЬКО в **Управление → Участники**.
- **S5-3 (item 3)** 👤 «Мои клубы» → «Мои заявки» → у pending-заявки **хорошо видимый** крестик «×» в правом-верхнем углу карточки (мягко-красный уголок). Тап → карточка **инлайн** превращается в «Отменить заявку? Нет / Отменить». **Ожид:** «Нет» — возврат к карточке; «Отменить» — заявка исчезает из «Мои заявки», тост «Заявка отменена». Повторно подать заявку в этот клуб снова можно (статус `cancelled` не блокирует).
- **S5-3b** 👤 Тап по телу карточки заявки (не по крестику) → переход на страницу клуба (как раньше). Крестик не триггерит переход.

## Регрессия (бегло — не сломалось ли смежное)
- **R1** Выход из клуба (free) работает; диалог с обязательствами показывается.
- **R2** Заявки: одобрить/отклонить (с причиной) работает; счётчик на табе «Клубы» корректен (только pending-инбокс).
- **R3** Карточка участника: кольца репутации, «Новичок»/организаторская рамка — как раньше.
- **R4** Создание события/складчины, отметка посещаемости — без регрессий.
- **R5** S1 в PAID-клубе: «Заметка» (всегда открытая) сохраняется своей кнопкой; ✎ → «Своя дата» сохраняется/закрывает карточку; подписочная полоса + «Взнос получен»/«Закрыть доступ» на месте — не сломаны.

### Что НЕ покрывается ручным тестом (backend/edge)
- Авто-экспирация: участник с истёкшей `subscription_expires_at` → шедулер роняет в `frozen` (hard cut). Требует прохода времени/правки даты в БД. Логика покрыта на бэке.
- 409-гонки (двойное «Взнос получен» из двух сессий) — обрабатываются silent-recovery.
