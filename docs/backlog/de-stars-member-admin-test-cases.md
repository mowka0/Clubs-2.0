## Предусловия (подготовка данных)
- **PAID-клуб** (open, цена > 0), вы организатор. Нужен ≥1 участник-`frozen` (👤 второй аккаунт вступает) и желательно ≥1 активный платный.
- **FREE-клуб** (цена 0), вы организатор, ≥1 участник.
- **CLOSED PAID-клуб** (по заявке, цена > 0) — для проверки одобрения заявки.
- Второй аккаунт (👤) для роли участника.

## A. Платный клуб — вступление honor-system (B10)
- **A1** 👤 Открыть PAID open-клуб → «Вступить». **Ожид:** экран «Вы вступили в клуб · доступ откроет организатор после взноса»; табов/контента нет; цена показана в ₽.
- **A2** 👤 Перейти по invite-ссылке в PAID-клуб → «Вступить в клуб». **Ожид:** «Добро пожаловать… доступ откроет организатор после взноса». Подписка в ₽.
- **A3** 👤 CLOSED PAID-клуб → «Хочу вступить» → отправить заявку. 🧑‍💼 На «Мои клубы» одобрить заявку. **Ожид:** участник становится `frozen` (появляется в «Ждут оплаты»), НЕ «ожидание оплаты Stars».
- **A4** 👤 FREE open-клуб → «Вступить». **Ожид:** сразу полноправный участник, контент/табы доступны (НЕ frozen).

## B. Цены в ₽ (B10/B11) — нигде нет Stars/⭐
- **B1** Каталог (Поиск) → карточка PAID-клуба. **Ожид:** «N ₽ / мес».
- **B2** Страница PAID-клуба, строка под названием. **Ожид:** «… · N ₽ / мес».
- **B3** Invite-страница PAID-клуба. **Ожид:** «Подписка · N ₽ / мес».
- **B4** Создать клуб → шаг «Участники». **Ожид:** лейбл «Цена подписки (₽/мес)»; при цене>0 подсказка «… до N ₽ в месяц. Участники платят вам напрямую — платформа комиссию не берёт» (без «80%»).
- **B5** Пройти все экраны выше. **Ожид:** слова «Stars» и значка ⭐ нет нигде.

## C. Дашборд участников — организатор (B9)
- **C1** 🧑‍💼 PAID-клуб → «Участники» (или Управление→Участники). **Ожид:** есть корзина «💸 Ждут оплаты» с frozen-участниками + кнопка «Взнос получен» в каждой строке.
- **C2** 🧑‍💼 Если есть участник с подпиской, истекающей ≤7 дней — корзина «⏳ Скоро закончится» с «до DATE · через N дней».
- **C3** 🧑‍💼 Спокойный список «Участники»: организатор (бейдж «Орг»), активные платные показывают «Активен · до DATE», у каждого — балл надёжности.
- **C4** 🧑‍💼 Нажать «Взнос получен» у frozen-участника. **Ожид:** тост «Взнос принят — доступ … продлён на 30 дней»; участник уезжает из «Ждут оплаты» в «Активные».
- **C5** 🧑‍💼 FREE-клуб → «Участники». **Ожид:** корзин «Ждут оплаты»/«Скоро закончится» нет, просто список (доступ-инфо не показывается).
- **C6** 👤 Обычный участник открывает таб «Участники» в клубе. **Ожид:** только активные, без корзин и без кнопок «Взнос получен».

## D. Карточка участника + гейт доступа (B9)
- **D1** Тап по участнику в списке → открывается карточка (кольца Надёжность/Посещаемость + bio + интересы).
- **D2** 🧑‍💼 В карточке активного платного: строка «Подписка активна до DATE · через N дней» + кнопки «Взнос получен · продлить до DATE» и «Закрыть доступ».
- **D3** 🧑‍💼 «Закрыть доступ». **Ожид:** тост «Доступ … закрыт»; участник становится frozen (в «Ждут оплаты»).
- **D4** 🧑‍💼 Карточка frozen-участника: «Доступ закрыт · участник ждёт оплаты» + «Взнос получен · открыть доступ» (кнопки «Закрыть доступ» нет).
- **D5** 👤 Обычный участник тапает по другому участнику. **Ожид:** карточка БЕЗ строки подписки и кнопок управления.
- **D6** 🧑‍💼 Тап по самому себе (организатору) в списке. **Ожид:** карточка без блока управления доступом (организатором управлять нельзя).

## E. Red-dot (item 2)
- **E1** 🧑‍💼 В клубе есть frozen ИЛИ истекающий ≤7д участник. **Ожид:** красная точка на табе «Управление» (страница клуба) и на табе «Участники» (экран Управление).
- **E2** 🧑‍💼 Подтвердить взнос всем frozen + нет истекающих. **Ожид:** точка гаснет (можно обновить экран).

## F. Кросс-клубовый «Ждут оплаты» на «Мои клубы» (item 1)
- **F1** 🧑‍💼 Иметь frozen-участников в ≥1 своём клубе → открыть «Мои клубы». **Ожид:** блок «💸 Ждут оплаты» (после «Заявки в мои клубы») со строками: имя (+@username), «{Клуб} · вступил(а) N назад», бейдж «Ждёт оплаты».
- **F2** 🧑‍💼 Тап по строке → карточка участника → «Взнос получен · открыть доступ». **Ожид:** доступ открыт, строка исчезает из «Ждут оплаты», тост.
- **F3** 👤 Аккаунт без своих клубов → «Мои клубы». **Ожид:** блока «Ждут оплаты» нет.

## G. Финансы (B11)
- **G1** 🧑‍💼 Управление → «Финансы». **Ожид:** только «Активных участников» + пояснение «Оплата подписок идёт напрямую вам, мимо платформы…». Нет «Выручка/Доля организатора/Комиссия платформы», нет «Stars».

## H. Складчина с frozen (CreateSkladchina)
- **H1** 🧑‍💼 PAID-клуб с frozen-участником → создать сбор → список участников. **Ожид:** frozen-участники задизейблены, тег «Ждёт оплаты», выбрать нельзя (в конце списка).
- **H2** 🧑‍💼 Активные участники выбираются нормально, сбор создаётся.

## I. «Мои клубы» — чистка awaiting (B8)
- **I1** Открыть «Мои клубы». **Ожид:** нет старых секций про оплату Stars («Ожидают оплаты»).
- **I2** «Мои заявки» — только pending-заявки; одобренная заявка в клуб → клуб появляется в «Где я состою» (для платного — как frozen, см. A3).

## J. S1 — админ-карточка: заметка + своя дата
- **J1** 🧑‍💼 Карточка участника → «✎ Редактировать». **Ожид:** появляется форма: поле «Своя дата окончания доступа» + «Заметка» + Сохранить/Отмена.
- **J2** 🧑‍💼 Поставить будущую дату → Сохранить. **Ожид:** карточка закрывается, тост «Изменения сохранены»; при повторном открытии «Подписка активна до» = новой дате; участник активен.
- **J3** 🧑‍💼 Написать заметку → Сохранить. **Ожид:** на карточке (read-only) блок «Заметка» с текстом. 👤 Обычный участник эту заметку НЕ видит.
- **J4** 🧑‍💼 ✎ → стереть заметку → Сохранить. **Ожид:** блок «Заметка» исчезает.
- **J5** 🧑‍💼 ✎ → дата: пикер не даёт выбрать прошлое (min = сегодня). (Бэк дополнительно отклоняет прошлую дату — 400.)

## K. СБП-реквизиты клуба (оплата взноса организатору, honor-system)
- **K1** 🧑‍💼 PAID-клуб → Управление → Настройки → ✅ есть секция «Реквизиты для взноса (СБП)» (ссылка/номер + подсказка). У FREE-клуба секции нет.
- **K2** 🧑‍💼 Задать ссылку + подсказку → Сохранить.
- **K3** 👤 frozen-участник этого клуба открывает клуб → ✅ под «доступ откроет организатор» кнопка **«Оплатить по СБП»** + текст подсказки. Тап → открывается ссылка организатора.
- **K4** 👤 Гость (не вступал) / заявитель на рассмотрении (приватный клуб, ещё не одобрен) → ✅ кнопки «Оплатить» НЕ видит (реквизиты только участникам).
- **K5** 🧑‍💼 Орг не задал реквизиты → ✅ у frozen-участника кнопки «Оплатить по СБП» нет (только текст про взнос).
- **K6** 🧑‍💼 Стереть ссылку в настройках → Сохранить → ✅ кнопка у участника пропадает.
- **K7** Подтверждение оплаты по-прежнему ручное: 👤 «оплатил по СБП» → 🧑‍💼 жмёт «Взнос получен» → ✅ доступ открыт (авто-подтверждения нет — это ожидаемо).

### K-NEW. Реквизиты обязательны при создании платного клуба (закрыли UX-дыру)
- **K8** 🧑‍💼 «+» → создать клуб → шаг «Участники»: цена = 0 → ✅ полей реквизитов нет. Поставить цену > 0 → ✅ появляется блок «Реквизиты для взноса (СБП)» с обязательным полем (★) + опц. подсказкой.
- **K9** 🧑‍💼 Цена > 0, поле реквизитов пустое → «Далее». **Ожид:** не пускает, ошибка «Для платного клуба укажите реквизиты для взноса». Вписать ссылку → «Далее» проходит, клуб создаётся.
- **K10** 🧑‍💼 Создать клуб с ценой > 0 + реквизитами → у frozen-участника сразу есть кнопка «Оплатить по СБП» (т.е. дыры из скриншота больше нет).
- **K11** 🧑‍💼 Настройки клуба: сменить FREE → платный (цена > 0) → ✅ появляются поля реквизитов; сохранить с пустой ссылкой нельзя (та же ошибка). Со ссылкой — сохраняется.
- **K12** 🧑‍💼 Платный клуб: в настройках стереть ссылку реквизитов → Сохранить. **Ожид:** не даёт сохранить (платный клуб обязан иметь реквизиты). *(Legacy: старый платный клуб без реквизитов при первом редактировании настроек попросит добавить ссылку — это ожидаемо.)*

### K-CLAIM. 2-шаговый флоу оплаты участником + заявка организатору (со скриншотом/наличные)
> Участник заявляет об оплате (СБП+скриншот ИЛИ наличные+чекбокс) → организатор видит заявку и подтверждает «Взнос получен» (honor-system, доступ открывает орг).
- **K13** 👤 frozen-участник платного клуба → на экране кнопка **«Оплатить взнос»** → тап открывает sheet «Оплата взноса» с суммой (💳 100 ₽ / мес).
- **K14** 👤 У клуба заданы реквизиты СБП → в sheet есть переключатель **«По СБП» / «Наличными»**. (Если реквизитов нет — только «Наличными», переключателя нет.)
- **K15** 👤 СБП: шаг 1 «Оплатить по СБП» (открывает ссылку орга) → шаг 2 «Прикрепить скриншот» (jpg/png ≤5МБ). **Ожид:** «Подтвердить оплату» неактивна, пока скриншот не прикреплён; после — активна.
- **K16** 👤 Прикрепить скриншот → превью → «Подтвердить оплату». **Ожид:** sheet закрывается, на экране клуба статус **«Оплата на проверке · по СБП»** (кнопки «Оплатить взнос» больше нет).
- **K17** 👤 Наличные: выбрать «Наличными» → чекбокс «Подтверждаю, что передал(а) взнос наличными» → «Подтвердить оплату». **Ожид:** статус «Оплата на проверке · наличными» (без скриншота).
- **K18** 🧑‍💼 Организатор открывает карточку этого участника (через «Ждут оплаты» или список). **Ожид:** блок **«⏳ Оплата заявлена · СБП/наличные»**; для СБП — кликабельный скриншот (открывается в полном размере); для наличных — пометка «скриншота нет».
- **K19** 🧑‍💼 На «Мои клубы» в блоке «Ждут оплаты» у заявившего участника бейдж меняется на **«Оплата заявлена»** (зелёный) вместо «Ждёт оплаты».
- **K20** 🧑‍💼 «Взнос получен» у заявившего → **Ожид:** доступ открыт, участник уходит из «Ждут оплаты», заявка (статус «на проверке» у участника) сбрасывается.
- **K21** 👤 Заявить об оплате можно только пока доступ закрыт (frozen). После открытия доступа кнопки/статуса оплаты нет.

## L. S2 — награды клуба (косметические награды организатора участнику)
> R2 «как интересы»: выбор эмодзи + название с автоподсказками из ранее созданных. R3: видны всем. R4: на репутацию НЕ влияют. Лимит 6, дубль по названию отклоняется.
- **L1** 🧑‍💼 Карточка участника → «✎ Редактировать». **Ожид:** в форме секция «Награды клуба» с кнопкой «＋ Добавить награду».
- **L2** 🧑‍💼 «＋ Добавить награду» → выбрать эмодзи (пикер ~12 шт.) + ввести название (≤40) → зелёная стрелка «→» (или Enter). **Ожид:** чип «эмодзи + название» появляется сразу; карточка НЕ закрывается (можно выдать ещё). Haptic на успехе.
- **L3** 👤 Второй аккаунт-участник открывает карточку этого же участника. **Ожид:** видит чип награды в секции «Награды клуба» (R3 — публично), но БЕЗ ✎ и без «×» (только чтение).
- **L4** 🧑‍💼 Выдать награду с тем же названием второй раз. **Ожид:** ошибка «Такая награда уже выдана» (дубль по названию отклонён, чип не дублируется).
- **L5** 🧑‍💼 Выдать 6 наград одному участнику. **Ожид:** после 6-й кнопка «＋ Добавить награду» исчезает, показывается «Максимум 6 наград»; 7-ю выдать нельзя.
- **L6** 🧑‍💼 ✎ → у чипа награды нажать «×» (снять). **Ожид:** чип исчезает; у второго аккаунта (L3) он тоже пропадает.
- **L7** 🧑‍💼 Автоподсказки: выдать награду «Активист» участнику №1. Затем у участника №2 → ✎ → «＋ Добавить награду» → начать вводить «Акт». **Ожид:** в подсказках появляется ранее созданная «Активист» (эмодзи+название); тап по подсказке выдаёт её сразу.
- **L8** 🧑‍💼 **FREE-клуб** (важно — попутный фикс): карточка участника → **есть** «✎ Редактировать» → форма с «Награды клуба» + «Заметка» (но БЕЗ «Своя дата окончания доступа», БЕЗ подписочной полосы и «Взнос получен» — это только платным). Выдать награду → работает. *(До S2 в free-клубе ✎ вообще не показывалась.)*
- **L9** 🧑‍💼 Тап по самому себе (организатору). **Ожид:** блока управления/наград нет (собой управлять нельзя).
- **L10** 🧑‍💼 R4-проверка: запомнить кольца репутации/уровень участника → выдать и снять награду. **Ожид:** репутация/уровень НЕ изменились (награды — чистая косметика).
- **L11** 🧑‍💼 Ошибка ввода: пустое название + «→». **Ожид:** кнопка «→» неактивна (выдать нельзя).

## Регрессия (бегло — не сломалось ли смежное)
- **R1** Выход из клуба (free) работает; диалог с обязательствами показывается.
- **R2** Заявки: одобрить/отклонить (с причиной) работает; счётчик на табе «Клубы» корректен (только pending-инбокс).
- **R3** Карточка участника: кольца репутации, «Новичок»/организаторская рамка — как раньше.
- **R4** Создание события/складчины, отметка посещаемости — без регрессий.
- **R5** S1 в PAID-клубе после реструктуризации формы: ✎ → «Своя дата» + «Заметка» сохраняются (блок J), подписочная полоса + «Взнос получен»/«Закрыть доступ» на месте — не сломаны добавлением наград.

### Что НЕ покрывается ручным тестом (backend/edge)
- Авто-экспирация: участник с истёкшей `subscription_expires_at` → шедулер роняет в `frozen` (hard cut). Требует прохода времени/правки даты в БД. Логика покрыта на бэке.
- 409-гонки (двойное «Взнос получен» из двух сессий) — обрабатываются silent-recovery.
