# Доверие к организатору + анти-скам платного клуба (дизайн, частично в беклоге)

> **Контекст:** de-Stars — платформа ВНЕ потока денег. Участник переводит взнос **напрямую** организатору (СБП P2P / наличные), платформа не держит и не возвращает деньги. Значит доверие к организатору критично, а классический эскроу/чарджбэк недоступен. Дата: 2026-06-27.

## 1. Решено к реализации СЕЙЧАС — карточка доверия организатора

На экране оплаты взноса (`DuesPaymentSheet`) — карточка «Кому вы переводите». Акцент на **аккаунте организатора**, метрик клуба нет. Каждый факт **условный, без нулей**.

- **Идентичность (всегда):** аватар + имя + @username + кнопка «Написать» (тап → `t.me/{username}`). Ядро доверия — можно проверить/написать живому человеку.
- **«На платформе с {дата}»** (всегда — это дата). Для свежего аккаунта → «недавно на платформе».
- **«Ведёт N клубов»** — только при N ≥ 2.
- **«Его клубам доверяют N участников»** (сумма активных по всем активным клубам орга) — только при N ≥ порога (старт: 5).
- **Nudge новичку:** если показывать нечего (свежий аккаунт) — мягкая подсказка «Организатор новый — напишите ему перед первым переводом» (тонкие данные → безопасное действие, НЕ блок).
- TG Premium — отложено (нужно начать сохранять `is_premium` из initData).
- Дата регистрации в TG — **не показываем** (только неточная оценка по `telegram_id`, риск показать неверное).

Макет: `docs/design/dues-payment/mockups/variants.html` (вариант A, 2 состояния: опытный/свежий).

## 2. Анти-скам: что НЕ работает (и почему)

- **«<5 участников → только наличные» / гейтинг метода оплаты — НЕ работает.** Платформа не в потоке денег ни в одном методе; гейт кнопки не гейтит перевод (реквизиты можно дать в личке). Наличные **хуже** для безопасности: нет даже скриншот-пруфа.
- **Завязка только на репутацию — не решает cold-start.** В t=0 честный новичок и скамер неотличимы (ноль на всех сигналах). Репутация защищает только состоявшихся оргов; новый аккаунт ею не закрыть — врождённое ограничение.

## 3. Что есть как «задел» (для будущего)

- **Account-credibility внутри L3-ранга клуба** — `ClubRankRepository.findProfileSignals` (owner-resistant per-account). Внутренний вход в ранжирование, за флагом, наружу не выводится. Можно тапнуть позже, но raw наружу не показывать (gaming).
- **Member-репутация/XP/уровень** орга как юзера (в своём клубе не копит — анти-фарм).
- ⚠️ Чистого «организатор надёжно открывает доступ за деньги» (delivery) score **нет** — его надо минтить (см. §4).

## 4. БЕКЛОГ (отдельная фича) — петля жалоб / не-доставки = репутация доставки + выпуск из cold-start

> Keystone анти-скама. Не просто «ловить скам», а **строить репутацию доставки** организатора по поведению, по которой новичок «выпускается» из ограничений cold-start.

**Идея:**
- У нас уже есть **claim** (`memberships.dues_claimed_at/method/proof`). Добавить отслеживание исхода: участник **заявил оплату → организатор открыл доступ?** в разумный срок.
- Сигналы «не-доставки»: claim висит N дней без «Взнос получен»; участник нажал **«Пожаловаться»** («оплатил — доступ не открыли»).
- Агрегируем по организатору → **delivery-репутация**: доля доставленных доступов, время до открытия, число подтверждённых жалоб.
- **Анти-скам:** при накоплении неразрешённых жалоб → метка «⚠ есть жалобы» на клубе/в карточке, авто-флаг/заморозка приёма новых платных вступлений, ручная модерация рецидивистов.
- **Cold-start graduation:** новый орг = «непроверенный» (бонусные ограничения, см. §5) → по мере роста delivery-репутации ограничения авто-снимаются, карточка получает позитивные факты.

**Объём:** таблица репортов/исходов claim, скоринг, шедулер, отображение в карточке + на клубе, модерация. Миграция (V≥42). Дизайнить отдельно.

## 5. Опционально (политика, не сейчас) — ограничение экспозиции на cold-start

- **Потолок цены для новых оргов:** пока не перешёл порог доверия (≥ N активных платящих ИЛИ ≥ M дней) — макс. цена ограничена (ограничивает ущерб на жертву). Минус: режет легит дорогие новые клубы → порог щадящий.
- **Информированное согласие под риск:** перед оплатой слабому оргу — предупреждение «платформа не возвращает деньги» + подтверждение. (Сейчас отвергнуто как отдельный шаг; частично закрыто nudge'ом в карточке.)

## Резюме
Репутация **награждает за поведение, а не предсказывает его** — поэтому cold-start закрывается не оценкой, а (а) честной карточкой «непроверен + контакт» сейчас и (б) петлёй доставки, которая даёт новичку заработать доверие и ловит скам, — потом.
