# Платное вступление — B+C, лёгкая версия (оплата → одно решение орга)

> **Решение PO (2026-06-28):** трилемма «A отсев-до-оплаты · B орг-подтверждает · C один гейт» → выбрано **B+C**.
> **Уточнение PO:** отдельная заявка/анкета на вступление НЕ нужна — переиспользуем уже собранный claim-флоу (оплата+скриншот), а «отсев» = добавить организатору у карточки участника кнопку **«Отказать · вернуть перевод»**.

## Суть (почти всё уже построено)

1. **Участник** (платный клуб): вступает → `frozen` → оплачивает (СБП+чек / наличные) → отмечает «оплатил» (claim). *(Уже есть.)*
2. **Организатор** у карточки участника видит профиль + карточку доверия + **скриншот оплаты** + статус → **одно решение:**
   - **«Взнос получен»** → доступ открыт (active, +30д). *(Уже есть.)*
   - **«Отказать · вернуть перевод»** → участник убирается из клуба (membership → cancelled), ему приходит уведомление «организатор отклонил вступление, вернёт перевод». Возврат — оффлайн, орг сам (honor-system, платформа вне денег). *(НОВОЕ — единственное, что добавляем.)*

«Отсев» в B+C = право отклонить **после** оплаты с возвратом (A пожертвовали). Кто платит — почти всегда реальный; отклонение редкое; орг видит чек.

## Что НЕ делаем
- НЕ добавляем отдельную заявку/анкету на платное вступление (PO: не нужно).
- НЕ трогаем applications для FREE closed-клубов (их анкета/одобрение — как есть).
- НЕ авто-возврат (платформа вне денег) — только уведомление + (опц.) напоминание оргу.

## Что строим (1 действие)
**Backend:** `POST /api/clubs/{clubId}/members/{userId}/reject-dues` (owner-only) — для `frozen`-участника: cancel membership + уведомить участника (refund-месседж). Опц. причина. Reuse `membershipRepository.cancel`.
**Frontend:** кнопка **«Отказать · вернуть перевод»** у карточки участника (рядом с «Взнос получен», для frozen). Тап → подтверждение → reject → участник убран, тост. На стороне участника клуб показывает «вступление отклонено» (membership cancelled).

## Открытые мелочи
1. Причина отказа — обязательна или нет? (рекомендую опционально; дефолт-месседж про возврат.)
2. Уведомление участника — DM ботом или просто статус «отклонено» на экране клуба? (минимум — статус; DM как плюс.)
3. После отказа повторное вступление возможно? (да — обычный join заново.)
