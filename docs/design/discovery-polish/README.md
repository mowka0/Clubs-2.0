# Полиш текущего главного экрана (Discovery)

**Статус: exploration, ничего не залочено.** Контекст (PO, 2026-07-21): полный
редизайн списка (`docs/design/discovery-list-redesign/`) отклонён, из него
понравилась только идея «полок со смыслом». Курс — докрутка текущего экрана.
Раздражители:

1. **Сетка метрик на карточке клуба** (`rd-mrow`/`rd-mstat`: серые коробки
   «88 дней / 3 участника / 100%») — выглядит чужеродно в обеих темах,
   не вписана в язык стекла.
2. **Плитки «Репутация / В клубах»** под шапкой — под вопросом: их место в
   профиле, а «Состоишь в N клубах» и так дублируется в подписи шапки.

Во **всех** вариантах ниже плитки статистики убраны (переезжают в Профиль),
шапка/поиск/чипы не тронуты. Варианты различаются только решением метрик
(+ полка в №3).

Мокапы: `mockups/*.html`, тема переключается `?light` в URL; рядом PNG обеих тем.

## ✅ ВЫБРАНО (PO 2026-07-21): `04-final.html`

Комбинация: текущий экран **минус плитки статистики** + полка **«Встречаются
сегодня»** (из варианта 3) + тройка метрик **тонкой стеклянной полосой** на нижнем
крае обложки (из варианта 2, но однострочной — число и подпись на одной базовой
линии, поэтому уже). Полоса лежит на тёмном скриме → одинакова в обеих темах.
Рендеры: `04-final-dark.png` / `04-final-light.png`. Это база для реализации.

## Заметки по реализации

- Удаление плиток = убрать блок `rd-stats` из `DiscoveryPage.tsx` (рендерится при
  `hasReputation`); репутация уже есть в Профиле (`pf-rep-list`), потери данных нет.
- Варианты 1–2 — только `ClubCard.tsx` + CSS, данные те же (`ClubCardFactsDto`).
- Полка «Встречаются сегодня» (№3) строится из уже приходящего
  `club.nearestEvent` по загруженным страницам выдачи — бэк не нужен, но полка
  видит только загруженные страницы (при пагинации 20 — практически всегда все
  клубы города). Полка скрывается, если таких клубов нет.
- Полка из №3 совместима с любым решением карточки (№1 или №2).
